# Deploy to Azure Static Web Apps

The user wants to publish their React site. This deploys to the Oxford Economics Azure subscription.

**Azure target:**
- Subscription: `5def3d89-62c5-4b4e-afea-c447e4b321f1`
- Resource group: `claude-resources`

## Step 1: Find the site directory

- If `$ARGUMENTS` is provided, use that as the site directory
- Otherwise, look for React projects (directories with `package.json` + `src/`) in the current working directory
- Confirm with the user which site to deploy

## Step 2: Check Azure CLI is installed

```bash
az --version
```
If not installed, tell the user: "You need Azure CLI. Install it from https://aka.ms/installazurecli or run `winget install Microsoft.AzureCLI`"

## Step 3: Authenticate to Azure

Check which auth method is available, in this order:

### Method 1: Service Principal (env vars)
Check if these environment variables are set:
```bash
echo $AZURE_CLIENT_ID
echo $AZURE_CLIENT_SECRET
echo $AZURE_TENANT_ID
```

If all three are set, log in with:
```bash
az login --service-principal \
  --username $AZURE_CLIENT_ID \
  --password $AZURE_CLIENT_SECRET \
  --tenant $AZURE_TENANT_ID
```

Tell the user: "Logging in using the service principal credentials your admin provided."

### Method 2: Interactive browser login
If no service principal env vars are set:
```bash
az login --tenant oxfordeconomics.com
```

This opens a browser. Tell the user: "A browser window will open — sign in with your Oxford Economics account."

### After auth, set the subscription:
```bash
az account set --subscription 5def3d89-62c5-4b4e-afea-c447e4b321f1
```

## Step 4: Build the React app

```bash
cd <site-directory>
npm run build
```
This creates the static output in `dist/`.

## Step 5: Ask for an app name

Ask the user for an app name (used in the URL). Suggest one based on the project folder name.
The URL will be `https://<auto-generated>.azurestaticapps.net`.

## Step 6: Create the Static Web App and deploy

```bash
# Create the Static Web App (if it doesn't already exist)
az staticwebapp create \
  --name <app-name> \
  --resource-group claude-resources \
  --location "westeurope" \
  --subscription 5def3d89-62c5-4b4e-afea-c447e4b321f1

# Get the deployment token
DEPLOYMENT_TOKEN=$(az staticwebapp secrets list \
  --name <app-name> \
  --resource-group claude-resources \
  --subscription 5def3d89-62c5-4b4e-afea-c447e4b321f1 \
  --query "properties.apiKey" -o tsv)

# Deploy the built output
npx @azure/static-web-apps-cli deploy ./dist \
  --deployment-token $DEPLOYMENT_TOKEN \
  --env production
```

## Step 7: Get the live URL

```bash
az staticwebapp show \
  --name <app-name> \
  --resource-group claude-resources \
  --subscription 5def3d89-62c5-4b4e-afea-c447e4b321f1 \
  --query "defaultHostname" -o tsv
```

## Step 8: Tell the user

- "Your site is live at https://<hostname>"
- "Anyone with the link can see it"
- "To update it, make your changes and run `/azure-site-builder:deploy-site` again"
- "Azure Static Web Apps free tier: 100GB bandwidth/month, 2 custom domains"

## Auth setup instructions (if user asks how to set up the key)

Tell them: "Ask your admin for three values and set them as environment variables:

**Windows (PowerShell):**
```powershell
$env:AZURE_CLIENT_ID = "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
$env:AZURE_CLIENT_SECRET = "your-secret-here"
$env:AZURE_TENANT_ID = "your-tenant-id"
```

**Or add them permanently** via System Settings → Environment Variables.

**Or create a `.env` file** in your project root (DO NOT commit this to git):
```
AZURE_CLIENT_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
AZURE_CLIENT_SECRET=your-secret-here
AZURE_TENANT_ID=your-tenant-id
```

Once set, the deploy command will use these automatically — no browser login needed."

Site directory: $ARGUMENTS
